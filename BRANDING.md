# Branding and Naming

Recommended repository name:

```text
community-codex-windows-troubleshooting
```

Recommended public title:

```text
Community Codex Windows Troubleshooting Field Guide
```

## Why This Name

- `codex windows troubleshooting` matches high-intent search queries.
- `Unofficial` reduces the risk that users mistake the project for an OpenAI-maintained resource.
- `Field Guide` communicates that the project is practical, evidence-based, and community maintained.

## Naming Options Considered

| Name | SEO/GEO | Brand-risk notes | Verdict |
| --- | --- | --- | --- |
| `community-codex-windows-troubleshooting` | Strong: exact troubleshooting query plus community qualifier | Best balance of discoverability, trust, and non-affiliation clarity | Recommended |
| `awesome-codex-windows-troubleshooting` | Strong: exact troubleshooting query plus "awesome" discovery pattern | Better if the repo is mainly a curated link list; weaker for a tested field guide | Acceptable but not preferred |
| `unofficial-codex-windows-troubleshooting` | Strong: exact troubleshooting query plus unofficial qualifier | Safest non-affiliation signal, but less warm and less community-oriented | Acceptable fallback |
| `codex-windows-troubleshooting` | Strong: exact troubleshooting query | Usable with clear disclaimer, but unofficial/community status is not visible in the repo slug | Acceptable fallback |
| `codex-windows-field-guide` | Good, but less direct search intent | Acceptable with disclaimer | Good title phrase, weaker repo name |
| `codex-windows-reliability-guide` | Good for engineering audience | Sounds broader than current troubleshooting scope | Possible later |
| `awesome-codex-windows-errors` | Searchable, but informal | "awesome" implies curated list rather than verified guide | Not recommended |
| `openai-codex-windows-guide` | Strong search | Higher confusion risk because it starts with OpenAI | Avoid |

## Brand Safety Rules

- Do not use OpenAI logos, icons, wordmarks, screenshots from official pages, or brand assets.
- Do not imply OpenAI affiliation, endorsement, sponsorship, certification, or official support.
- Keep `Unofficial` visible in the README title and repository description.
- Use "Codex" and "OpenAI" only descriptively to identify the product and upstream sources being discussed.
- Prefer "Codex on Windows" or "Codex Windows troubleshooting" over product-like names such as "Codex Windows Support".
- If a logo is added later, it must be original and must not resemble OpenAI branding.

Official reference: [OpenAI brand guidelines](https://openai.com/brand/).
